تمام يا أبو محمد! جهّزت لك “برومبت نصّي” متكامل للذكاء الاصطناعي علشان يبني ويملأ صفحة «المراسل» في سبـق الذكية — مع سكيمة JSON واضحة، ونسخة UI/UX جاهزة حسب معاييرنا (Next.js 15 + Tailwind + Radix UI + Lucide). انسخه وحطّه في وكيل الـAI عندك وخلّه يرد بالـJSON المطلوب.

⸻

برومبت ذكاء اصطناعي: صفحة المراسل (Sabq AI CMS)

الدور المطلوب من الذكاء الاصطناعي:
أنت مساعد تحرير وبيانات لصفحة «بروفايل المراسل» في نظام Sabq الذكية. مهمتك جمع/تلخيص بيانات المراسل وتوليد نصوص واجهة عربية احترافية + مخرجات JSON بالهيكل المحدد أدناه فقط (دون أي شرح خارج JSON).

المدخلات (Inputs)
	•	reporter_id أو reporter_slug
	•	نافذة زمنية للإحصاءات: stats_window_days (افتراضي 90)
	•	عدد آخر المقالات: last_articles_limit (افتراضي 5)

المطلوب (Tasks)
	1.	ملف المراسل: الاسم الكامل، الصورة، المنصب/المسمى التحريري، النبذة، شارة التوثيق.
	2.	مؤشرات رئيسية (KPIs):
	•	إجمالي المقالات المنشورة
	•	إجمالي المشاهدات لكل المقالات
	•	إجمالي الإعجابات لكل المقالات
	•	متوسط زمن القراءة (دقيقة)
	•	معدل الإكمال (٪) إن توفر
	3.	آخر المقالات (limit=5): لكل مقال (العنوان، الرابط/الـslug، التاريخ، التصنيف+الوسم اللوني، عدد المشاهدات، الإعجابات، التعليقات، وقت القراءة، حالة «عاجل»).
	4.	التصنيفات الأعلى للمراسل: أعلى 5 تصنيفات كتب فيها مؤخراً (الاسم، slug، نسبة المساهمة، إجمالي مشاهدات تلك الفئة).
	5.	سلسلة زمنية خلال stats_window_days: مجموع المشاهدات اليومية، مجموع الإعجابات اليومية (لرسم مخطط بسيط).
	6.	شارات / Badges (اختياري): مثل “موثق”، “الأكثر نشاطاً هذا الشهر”، “متخصص في (رياضة/اقتصاد/…)” — منطقية وغير مبالغ فيها.
	7.	نصوص واجهة قصيرة (Arabic RTL، لهجة صحفية احترافية):
	•	سطر ترحيبي بالمراسل (Headline)
	•	سطر فرعي يوضح تخصصه ونطاق تغطيته
	•	CTA موجز: “تابع تغطياته” / “اطلع على آخر تقاريره”
	8.	حالات عدم توافر بيانات:
	•	لا توجد مقالات حديثة → اقترح نص بديل محايد + أعد last_articles كمصفوفة فارغة.
	•	مراسل غير موثق → لا تضف شارة التوثيق.
	•	حساب غير نشط → اجعل كل القيم 0 وأرسل status: "inactive".

معايير الكتابة والتجربة (UX/UI)
	•	لغة عربية واضحة، موجزة، دون مبالغة.
	•	دعم RTL.
	•	العناوين ≤ 60 حرفاً، الأسطر الفرعية ≤ 120 حرفاً.
	•	لا تُرجع أكواد أو HTML — أعد JSON فقط.

ملاحظات تنفيذ الواجهة (لك يا فريقنا ✨)

بنية الصفحة (Next.js 15 /app/reporter/[slug]/page.tsx)
	•	Header: صورة شخصية (Avatar), الاسم, المسمّى, شارة التوثيق (Radix Badge), أزرار متابعة/مشاركة.
	•	KPI Grid (4–5 بطاقات صغيرة): إجمالي المقالات، إجمالي المشاهدات، إجمالي الإعجابات، متوسط وقت القراءة، معدل الإكمال.
	•	Section: آخر 5 مقالات (Card list) مع شارة «عاجل» (Badge أحمر) وChip التصنيف.
	•	Section: التصنيفات الأعلى (Bar mini-cards) مع نسبة المساهمة.
	•	Section: المخطط الزمني البسيط (Recharts LineChart) للمشاهدات/الإعجابات (toggle).

Tailwind (مقتطف Tokens)
	•	خلفية داكنة خفيفة على الهيدر: bg-neutral-900 text-white (أو حسب SmartThemeApplicator)
	•	بطاقات KPIs: rounded-xl border border-neutral-200/60 dark:border-neutral-800 p-4
	•	شارات:
	•	verified: bg-emerald-600/10 text-emerald-600
	•	breaking: bg-red-600 text-white
	•	Chips التصنيف: inline-flex items-center gap-2 rounded-full px-3 py-1 text-sm

Radix UI + Lucide (اقتراحات)
	•	Badge, Avatar, Tooltip, Tabs
	•	Icons: User, BadgeCheck, Eye, Heart, MessageSquare, Flame, BarChart3, Trophy, Newspaper

عقد API المقترح (Server Actions/Route)
	•	GET /api/reporters/:slug/overview?days=90&limit=5
	•	يجمع: الملف الشخصي، KPIs، آخر المقالات، التصنيفات الأعلى، timeseries.
	•	فهارس DB تدعم الاستعلام بسرعة (author_id + published_at DESC، views_count DESC).

Prisma (استعلامات مختصرة توجيهية)
	•	findUnique(reporter) + join على articles المنشورة خلال النافذة الزمنية.
	•	groupBy للتصنيفات (count, sum views).
	•	aggregate للمجاميع (views/likes) ومتوسطات (reading_time).

حالات حافة
	•	بدون مقالات حديثة: اخفِ مخطط timeseries، وأظهر رسالة هادئة: “لا توجد تغطيات حديثة، اطلع على أرشيف المراسل.”
	•	غير موثق: لا تُظهر Badge التوثيق.
	•	غير نشط: عطّل CTA وأظهر تنبيه رمادي صغير.
